How to update your Ghost blog

16 May 2016 on ghost, technology, git

Since this is the first time I'm updating my Ghost-based blog, I want to share the process I followed in order to do it successfully. Even though it's purely a manual process, I hope that an automatic update feature gets implemented soon as part of the Ghost's next releases in the near future.

The Backup

Before you attempt anything, I strongly suggest you backup your Ghost blog folder in case something goes wrong (I've been there too :S). By the way, it's always a great idea to keep a repository of your blog files. I keep mines on GitHub but you could use also Bitbucket.

Let's update that $#*!

Download Ghost's latest release and extract the zip file to a temporary location or you could simply clone it from GitHub. Open your Terminal and type:

git clone git://github.com/tryghost/ghost.git

This will create a ghost folder at your current local file system location. If you want to specify a different folder name type:

git clone git://github.com/tryghost/ghost.git your_folder_name

Next, navigate to your original Ghost blog folder and delete the core and node_modules folders. Keep the content folder. You can also delete all the *.js (except config.js), *.json and *.md files.

Now, navigate to the ghost folder you just downloaded. In case you have cloned it from GitHub, first you have to switch to the stable branch (production environment) by typing git checkout stable in the Terminal.

Inside that folder move the core folder and all the *.js, *.json and *.md files to your original Ghost blog folder. If you're using the Casper theme included with Ghost, you may move the content/themes/casper folder to your content/themes folder.

Open the Terminal, navigate to your original Ghost blog folder and type:

npm install --production

This will create a node_modules folder to include all the dependencies of your Ghost blog. In case you want to check if the whole update process went successful, first install the remaining devDependencies by typing:

npm install

And start the server in development mode with:

npm start

Open a web browser at http://localhost:2368 to verify Ghost is up and running. After this you can deploy your Ghost blog as usual to your server. In my case, I'm using Heroku and it's as simple as typing:

git push heroku master

Congratulations! You have updated your Ghost blog successfully and now you can continue writing awesome posts as usual. Until next time.
